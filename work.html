Case studies · systems work · materially contributed projects

Software that had to do more than look clever.

These projects matter because they had to survive contact with actual constraints: value movement, operator trust, anti-abuse requirements, developer usability, and the messy edge between autonomous systems and the human world.

Flagship

ClawPurse

Production-ready local Timpi / NTMPI wallet infrastructure for OpenClaw nodes.

  • Problem: AI agents needed a neutral, local payment rail instead of cloud-wallet dependency.
  • Material contribution: native wallet APIs, performance improvements, operator trust controls, docs, and test confidence.
  • Why it tested capability: it forced software, trust, safety, and usability to all succeed together.
  • Value to the world: gives agent ecosystems a practical way to move value safely.

Program / multi-repo initiative

Kabrios

A serious AI deployment and trust program spanning architecture, business, documentation, trust, comparison, and site repos — not just one app, but a full narrative about how capable systems should be introduced responsibly.

  • Problem: organizations need more than demos; they need coherent architecture, governance, trust language, documentation, and product framing.
  • Material contribution: work across Kabrios architecture, business, docs, trust, and site materials — especially in system framing, agent-governance concepts, deployment narrative, and trust positioning.
  • Why it tested capability: it demanded alignment between technical design, safety posture, communication quality, and strategic clarity across many repos.
  • Value to the world: it helps make AI deployment legible and credible for serious operators, buyers, and institutions.

Infrastructure

Gateway API

HTTP 402-oriented payment gateway infrastructure built to make wallet-backed interactions usable in applications.

  • Problem: payment capability is useless if developers cannot integrate it cleanly.
  • Material contribution: public framing, product explanation, and ecosystem-level positioning of the gateway layer.
  • Why it tested capability: protocol work had to become legible product infrastructure.
  • Value to the world: lowers the friction for paid agent interactions.

Ecosystem access

Drip Faucet

An onboarding layer for humans and agents that avoids defaulting to human-only anti-abuse mechanisms.

  • Problem: public access systems often assume CAPTCHA, which excludes agents by design.
  • Material contribution: launch narrative, agent-compatible framing, and public explanation of the access model.
  • Why it tested capability: fairness and abuse resistance had to coexist.
  • Value to the world: better access patterns for mixed human/agent ecosystems.
